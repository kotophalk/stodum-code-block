# StoDum Code Block

Легкий, быстрый и современный плагин для вывода блоков кода в редакторе Gutenberg (WordPress) с подсветкой синтаксиса на базе Highlight.js.

Плагин разработан с фокусом на нулевую раздутость (zero-bloat) и высокую производительность фронтенда. Включает в себя встроенный инструмент для массовой или точечной миграции старых блоков (например, штатных `core/code` или `core/preformatted`) в новый формат.

## ✨ Ключевые возможности

- **Gutenberg-First**: Нативный блок `stodum/code-block` с полной поддержкой визуального редактора и Server-Side Rendering.
- **Подсветка синтаксиса**: Интеграция с Highlight.js (загрузка через CDN) без тяжелых зависимостей и конфликтов с другими JS-библиотеками.
- **Инструмент миграции**: Встроенный интерфейс (в разделе "Инструменты -> StoDum Code Block") для сканирования всей базы статей и миграции старых блоков с "умным" определением языков команд (bash, docker, python и др.).
- **Smart Heuristics (v1.0.6)**: Интеллектуальное определение языка даже без использования Markdown-тегов. Плагин "узнает" PHP по переменным и функциям (`$`, `->`), Bash по командам (`docker`, `sudo`) и JavaScript по современному синтаксису.
- **Swift Guard**: Уникальная система защиты от ложного срабатывания автоопределения Highlight.js (предотвращает ошибочное превращение любого кода в Swift).
- **Синхронное переключение тем**: Фронтенд-поддержка переключения светлой/темной темы кода для всех блоков на странице одним кликом с сохранением выбранного цвета фона.
- **Поддержка множества цветовых палитр**: Доступен широкий выбор палитр (Atom One, GitHub, Monokai, Dracula, Nord, VS Code и др.).

## 🏗 Архитектура плагина

Кодовая база полностью реорганизована с использованием модульного ООП-подхода для простоты сопровождения:

- `stodum-code-block.php` — Точка входа. Ультра-легкий загрузчик, который занимается исключительно регистрацией ассетов и рендерингом Gutenberg-блока (`render_block`).
- `includes/class-stodum-settings.php` — Отвечает за панель управления настроек в WP Admin и рендеринг таблицы интерфейса мигратора.
- `includes/class-stodum-migrator.php` — Ядро бизнес-логики: парсинг блоков (`parse_blocks`), эвристики для определения языков, безопасные AJAX-обработчики и изменение записей в базе данных (`wp_update_post`).
- `assets/` — Vanilla JS и CSS для фронтенда блока, а также скрипты админ-панели (мигратор, тостер-уведомления).
- `blocks/code/` — Исходный код React/ESNext для блока редактора Gutenberg.

## 🛠 Технический стек и стандарты

- **PHP 7.4+**: Строгая типизация, использование статических методов-обработчиков (Singular Entrypoint). Безопасность обеспечена проверкой nonce-отпечатков (`check_ajax_referer`) и разрешений (`current_user_can`).
- **JavaScript (Vanilla)**: Нулевая зависимость от jQuery на фронтенде. Взаимодействие с DOM реализовано через современные браузерные API.
- **Хранение данных**: Плагин **не создает** дополнительных таблиц в БД. Все данные блоков хранятся в стандартном формате комментариев Gutenberg.
- **Производительность**: Ассеты (CSS/JS) подключаются *только* в том случае, если на странице присутствует хотя бы один блок кода (Lazy Enqueueing).

## 🚀 Использование мигратора

Для перевода существующего сайта с блоками `core/code` или `core/preformatted` на StoDum Code Block:

1. Перейдите в **Инструменты → StoDum Code Block**.
2. В блоке "Code Block Migrator" нажмите **Scan Posts**.
3. Плагин найдет все статьи, где присутствуют устаревшие блоки кода, и покажет их в виде таблицы.
4. Вы можете воспользоваться кнопкой **Preview**, чтобы посмотреть результаты "до/после", или нажать **Migrate** у каждой статьи отдельно. Кнопка **Migrate All Found Posts** сконвертирует все найденные блоки за один проход.

## 📝 Разработка и участие

Если вы хотите расширить функциональность блока (например, добавить новые языки или темы):

1. Цветовые темы хранятся в статическом реестре `StoDum_Code_Block::get_theme_registry()`. Для добавления новой темы достаточно указать ее `dark_css` и `light_css` из официальных файлов Highlight.js.
2. Логика эвристического определения языка по первой строке (если язык не был задан вручную) находится в `StoDum_Migrator::convert_single_block()`.
